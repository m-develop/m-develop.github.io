    Ant-Tool

    Dear all :

    在工具共建小组近 5 个月的持续迭代下，在今天 ant tool 终于达到了 1.0 发布需求。借此机会我们再来向大家介绍下 ant tool。

    ant tool 是一套更加面向社区的、更加轻薄的解决前端构建和调试的通用型脚手架方案。

    在 ant tool 体系下有很多下沉至开源社区的模块，而这些模块具备了：构建、调试等所有功能。而从点及面这个过程由具体脚手架实现，可以说脚手架是某个业务场景下的最佳业务实践，也是工具在特定业务中的最佳实践。所以 ant tool 只是一个 代号，它并不存在 cli，所有的入口我们都收敛为如 tnpm/npm run build 等。

    现状

    目前 ant tool 体系已经完全覆盖了本地开发所需要的一切要素，通过脚手架工具，只需几个命令，便可轻松搭建一个项目或组件框架，随之而来的本地构建，本地或在线调试，无线模拟器调试，无线容器调试，自定义构建，自定义调试等等都会了然于眼前。

    在这边也很兴奋的告诉大家，据不完全统计目前已有上百个项目已经使用了这套体系。

    针对用户

    在用户层，用户只需要针对自己业务的形态选择对应的脚手架工具即可。在每个脚手架工具中都有对应业务的脚手架，而脚手架中已经描述了工具在该特定业务中的最佳实践。

    没错，作为用户你只需要知道：

    • antd 在 sofa 下开发 => 使用 antd-init
    • 无线端开发 => 使用 m-init
    • 通用组件开发 => 使用 component-init
    • antd 在 chair 下开发 => 使用 chair-react

    针对开发者

    atool-build

    它是对 webpack 的进一步封装，它会为你默认生成一套配置文件，该配置文件中包含了大部分常用的 webpack loader 和 plugin，最终调用 webpack 进行构建。需要自定义构建配置时，atool-build 允许用户指定构建所用的配置文件。

    dora

    dora 的名字取自哆啦A梦(doraemon)，dora 的插件则象征着 doraemon 四维口袋里的各种未来道具。 dora 是一个开发服务器，通过插件的方式集合各种调试方案，比如 webpack、livereload、browsersync、数据 mock、本地代理、weinre、jsonapi 等等。目前 dora 已经拥有 14 个插件，也得益于插件化的设计，pc 到 无线只需要添加对应插件即可。

    atool-test

    atool-test 是专注于写测试的工具，他所需要的配置非常简单。 在 atool-test 中默认内置了 mocha chai sinon ，并和 atool-build 和 dora 实现互通。

    atool-doc

    atool-doc 他是一个面向组件的静态 Demo 生成、调试工具。某种意义上它具有调试代码的作用。

    moggles

    moggles 的名字取自游戏 Don’t Starve 中的道具，拥有了他你可以看清整个黑暗的世界。 moggles 的出现正含有此寓意，在离线包的开发中，其实内藏了很多很多的黑盒，这些黑盒来源于不同的容器，来源于不同的加签服务，来源于不同的部署平台，来源于不同的模拟器调试需求等等。他的出现就是把这些作为开发者不需要知道东西抹平。

    moggles 的优势或许有： 效率：支持并发构建多个支付系 amr 离线包，或者非支付系 zip 包。 流程：完美契合 Dora 和 atool-build, 完美对接云部署平台。
