    初始化项目

    通过m-init初始化项目，可以参考初始化教程

    完善配置

    • 补全package.json的相应信息

      • appid：产品的appid
      • name：系统名称，是一个英文单词，比如lendweb
      • description：项目名称，通常是中文，比如借条

    • 配置lengjing.config.js

      • 一般只需要补全页面以及页面名称即可
    var pkg = require('./package.json');

module.exports = {
  appid: pkg.appid,
  name: pkg.name,
  desc: pkg.description,
  host: `https://${pkg.appid}.h5app.alipay.com/www/`,
  pages: [
    {
      entry: 'index.html',
      desc: '首页'
    },
    {
      entry: 'detail.html',
      desc: '项目详情'
    }
  ]
};

    tnpm run init

    执行tnpm run init之后，会根据你配置的lengjing.config.js生成你需要的项目文件，包括html、less以及js文件，并配置好package.json的entry。

    开发业务

    通过tnpm run init命令之后，就可以动手开发业务逻辑了，下面以开发一个首页为例。

    1. 生成了index文件夹，结构如下
    ├── index.atpl // 可以自己按需添加，一个简单的js模板
├── index.html
├── index.js
├── index.less

    我们提倡以业务逻辑定义文件夹，在该文件夹内包含该业务所需要的所有文件，在webpack的帮助下，以js为核心管理所有的资源。

    首页代码示例：

    <!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8"/>
  <title>anima</title>
  <meta name="apple-mobile-web-app-capable" content="yes"/>
  <meta name="apple-mobile-web-app-status-bar-style" content="black"/>
  <meta name="format-detection" content="telephone=no"/>
  <meta name="format-detection" content="email=no"/>
  <meta name="viewport" content="width=device-width,initial-scale=0.5,maximum-scale=0.5,minimum-scale=0.5,user-scalable=no">
  <script src="common.js"></script>
  <script src="vw.js"></script>
  <script type="text/javascript">vw(32);</script>
  <link rel="stylesheet" href="common.css"/>
  <link rel="stylesheet" href="index.css"/>
</head>
<body>
<div class="index-main" id="main">首页</div>
<script src="index.js"></script>
</body>
</html>

    关键点：

    • anima-hd作为高清方案，通过<script src="vw.js"></script>加载对应的高清模式，主要为flex和vm两者，具体可以查看anima-hd介绍
    • 整个页面的逻辑全部通过index.js来完成，需要在package.json的entry配置对应的entry，除了common.js，基本上每个html访问的js资源都有一个对应的entry配置

    通过tnpm run init生成的页面，已经自动配置entry

    "entry": {
    "vw": "./src/common/vw.js",
    "index": "./src/index/index.js"
}
    • index.html生成之后的路径默认在根目录下，通过http://localhost:8001/index.html访问

    首页JS代码示例：

    /**
 * @file index
 * @author anima
 */
import './index.less';
import './index.html';
import util from '../common/util';
import { $ } from '../common/base';
import render from './index.atpl';

const page = {
  init() {
    util.getRPC('alipay.pcredit.huabei.prod.main', {test: '测试参数'}, (res) => {
      const html = render({
        data: res.data
      });
      $('#main').html(html);
    });
  }
};

page.init();

    关键点：

    • 以ECMAScript 6作为js开发版本，通过babel来转成ECMAScript 5，如果你对此不熟悉，可以访问ECMAScript 6 入门来学习。
    • 默认给.atpl扩展名的文件提供了atpl-loader，加载index.atpl会变为一个render函数，通过render(data)方式就可以渲染模板得到HTML片段
    • 通过import less文件，webpack会自动进行编译，生成对应的同名css文件，同时不要忘记import html，否则HTML页面无法在根目录下访问。

    使用AntUI

    anima-boilerplate使用anima-hd作为默认的高清方案,默认使用100px作为rem的基准，而AntUI是使用32px为基准，所以在使用AntUI 组件时候，需要手动设置。

    // index.html
<script src="vw.js"></script>
<!-- 适配antui，手动设定32-->
<script type="text/javascript">vw(32);</script>

// index.less
.title {
  font-size: 30 * @ppx; // 在640分辨率下通过less计算为30px, @ppx为0.03125rem
}

    这样就能在设计稿和AntUI使用同一个rem单位。

    首页的AntUI使用效果图

    使用common

    • 默认每个entry都需要加载common/base, 这样webpack才能打包进common.js
    • 推荐通过import {$} from '../common/base' 来调用anima-yocto, 保证每个entry调用common/base

    使用mock

    anima-boilerplate提供针对状态的mock方案，通过结合mtracker可以方便的对每个状态进行管理和验证。详情可以查看Mock数据

    如mock index.js示例中的alipay.pcredit.huabei.prod.main请求

    util.getRPC('alipay.pcredit.huabei.prod.main', {test: '测试参数'}, (res) => {
    console.log(res);
});

    主要分为几步

    • 在mock/mockData添加alipay.pcredit.huabei.prod.main为名字的js文件
    • 在上述文件内编写数据内容
    • 通过util.getRPC调用对应的mock文件，获取mock的数据

    下面就是mock/mockData/alipay.pcredit.huabei.prod.main.js示例

    export default function (viewId) {
  const mock = {
    default: {
        name: 'anima'
    },
  };

  return mock.default;
}

    可以看到改文件返回的是一个mock function，我们在mocGetJSON会调用它，然后作为给util.getRPC的callback的入参，如上面的res就是{name: 'anima'}

    构建和打包

    通过上面的步骤，我们可以完成一个功能的开发，之后我们需要做的就是构建和打包。

    整个过程可以简单描述为

    src源码 ---> ant-build ---> 静态资源---> moggles ---> .amr (钱包离线包文件)
                                                    .zip (淘宝离线包文件)
    1. ant-build会将 es6转为现在主流浏览器能识别的es5，less转为css 等一系列资源的处理，转换之后所有生产环境的代码在静态资源目录，默认为www
    2. moggles将www内的文件根据各个平台的要求打包生成相应格式的压缩文件包，如钱包专用的amr文件，和现在手淘专用的zip文件
    3. 两者文件默认会根目录下的_package目录，以渠道+appid为文件夹名

    _package如下所示

    .
├── afwealth-20000199
│   └── 20000199_1_0_4_38_TAR.amr
├── deploylist.json
├── portal-20000199
│   └── 20000199_1_0_4_35_TAR.amr
└── taobao-20000199
    └── 20000199_1_0_4_33.zip

    部署上线

    在生成离线包之后，就可以通过部署和发布环节所讲的进行上线操作。

    接入棱镜

    脚手架提供了tnpm run lengjing的指令，发布完离线包之后，执行tnpm run lengjing即可立即接入棱镜做产品的分析。

    棱镜的地址：https://lengjing.alipay.com
